# 法務ドキュメント (社内ドラフト)

> **取り扱い区分**: Confidential / Internal Only
> **外部公開**: 現時点では **不可**。公開可否は管理部門・法務責任者・経営会議の承認を経て別途決定する。
> **配布範囲**: 経営層・管理部門・開発責任者・事業責任者のみ

本ディレクトリは STEP（以下「本サービス」）を有償提供するにあたり、法令・
ガバナンス・セキュリティ統制の観点から必要となる**上場企業水準**の内部文書を
ドラフトとして保管する。各文書は法務・監査・経営会議でのレビューを経た後、
必要に応じて対外版（公開版）として再整形する前提で作成している。

## 文書管理ルール

### 1. 改訂プロセス
1. 起案（法務または担当部門）
2. 関連部門レビュー（法務・セキュリティ・開発・経営）
3. 顧問弁護士レビュー（対外公開文書・契約関連文書は必須）
4. 経営会議承認
5. 版管理（各ファイル末尾の「改訂履歴」に記録）
6. 周知・教育（全社 or 対象部門）

### 2. 版管理
- 各文書の冒頭に「版」「制定日」「最終改訂日」「承認者」を明記
- 重要変更は顧客に事前通知（対外公開文書）
- Git 履歴を一次証跡、社内ドキュメント管理システム（別途構築）で版を公式記録

### 3. 監査・棚卸し
- 年 1 回（事業年度開始時）に全文書を棚卸しし、実態と乖離がないかを確認
- 内部監査室（設置後）の監査対象

### 4. 準拠法令・参照規格
- 個人情報保護法（APPI、令和 4 年改正）
- 特定商取引に関する法律
- 電子計算機使用詐欺罪・不正アクセス禁止法
- 消費税法（適格請求書等保存方式 = インボイス制度、令和 5 年 10 月施行）
- 会社法（計算書類・内部統制）
- 金融商品取引法（J-SOX、内部統制報告制度）
- GDPR（EEA 居住者を顧客に含む場合）
- ISO/IEC 27001（情報セキュリティマネジメント）
- ISO/IEC 27701（プライバシー情報マネジメント）
- JIS Q 15001（個人情報保護マネジメント、プライバシーマーク基準）
- SOC 2 Trust Services Criteria（将来的な取得想定）

## 免責

本ディレクトリ内のドキュメントは、執筆時点の法令・ガイドライン・プロダクト
仕様に基づく**ドラフト**であり、法的効力を保証するものではない。対外公開・
契約締結に用いる際は**必ず顧問弁護士のレビューを経ること**。
